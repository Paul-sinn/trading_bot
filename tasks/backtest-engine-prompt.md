## 왜 필요한가

거래기록도 백테스트도 없어 **엣지가 있는지, Kelly 입력(승률·손익비)이 무엇인지** 알 수 없다.
백테스트 엔진은 ① 헌장 전략의 엣지를 측정하고 ② 그 산출(승률·손익비·표본수)을 Kelly 콜드스타트
prior로 공급한다(`tasks/kelly-fix-prompt.md`와 연결). **1순위 작업.**

## 4. 구현할 전략 (헌장 반영 — 기존 코드와 다름!)

> ⚠️ 기존 `signals.py`의 "EMA/RSI/MACD 다수결"은 **폐기**한다. 헌장 §1이 그 혼합을 명시적으로 정리했다.

**① 모멘텀/추세 신호 (signals.py 재설계)** — 시계열 추세추종.
- 일봉 상승추세 판정(예: 가격 > 50d/200d, 추세 방향). SPY 대비 상대강도 필터.
- RSI는 *독립 매수신호 아님* → "상승추세 안의 눌림 타이밍"으로만 사용(헌장 §1 패러다임 해소).

**② 레짐 (regime.py 신규)** — 헌장 §8. SPY 200일선 + VIX → 4레짐:
```
A 정상강세 (SPY>200d, VIX<20)   : 풀 가동
B 불안강세 (SPY>200d, VIX20~30) : 사이징 축소, 고확신만
C 약세    (SPY<200d)            : 신규 진입 정지, 청산·축소
D 패닉    (VIX>30)              : 신규 전면 정지, 100% 디레버리지
```
레짐은 **마스터 스위치** — 종목 신호 위에서 진입게이트·사이징을 조절. C/D면 진입 불가.

**③ 진입 — 눌림목 주력 (entry.py 신규)** — 헌장 §1.
```
게이트: 일봉 상승추세 + SPY 상대강도 + 레짐 A/B
트리거: 추세 유지 중 단기 조정 후 재개 (예: 20d선 눌림 / RSI 리셋 후 반등)
체결  : 신호는 일봉 종가, 체결은 다음날 시가 (미래참조 금지!)
```
- **돌파(20일 Donchian 신고가)도 구현**해 눌림목과 **A/B 비교**(헌장이 데이터로 정하라 함).

**④ 청산 — R 기반 스케일아웃 래더 (exits.py 신규)** — 헌장 §7-2. 7단:
```
① 초기 ATR 스탑(전체) ② +1R 손절선 상향(완전 본전 아닌 구조 아래)
③ +1.5~2R 부분익절(소량) ④ 나머지 트레일링 ⑤ 레짐 깨짐 %청산(C:X%/D:100%)
⑥ 실적 전 개별주 축소·청산 ⑦ 타임 스탑(N일 무진전)
```
- **⚠️ 레이어로 검증**: 베이스라인(①+④) → +②(본전) → +③(부분익절) → +⑤⑦. **각 레이어가 Sharpe 개선하며 총수익 안 죽일 때만 채택.** ②③은 변동성↓·기대값↓ 트레이드오프라 **A/B 필수**.
- 모든 레벨은 R(초기 리스크) 배수로. ⑤는 **% 룰로 결정론화**(재량 금지).

**⑤ 사이징 (sizing.py)** — 헌장 §6·§7. **MDD가 governor.** 분수 켈리(켈리 입력은 백테스트 산출).
레짐별 사이징 배수 적용(A 풀, B 축소). 콜드스타트는 `kelly-fix-prompt.md` 따름.

## 5. 엔진 핵심 요구사항

**① 미래참조(look-ahead) 금지 — 최우선 정확성 요건**
- 바 `t` 의사결정은 `t`까지 데이터만. 신호=일봉 종가, **체결=다음날 시가**(같은 날 종가→같은 날 체결 금지).
- 지표 워밍업 구간(200d MA 등 안정화 전) 거래 제외.
- **검증 테스트 필수**: 미래 가격을 바꿔도 과거 의사결정 불변.

**② 비용·슬리피지 (헌장 §10 정직성 규칙)**
- 진입/청산마다 슬리피지(bps) + 수수료 차감. **보수적으로**(일봉은 체결 낙관 쉬움 → 부풀림 방지).
- Robinhood 무수수료라도 PFOF 슬리피지 반영(기본 슬리피지 0 금지).

**③ 데이터 (헌장 §3 — 무료 일봉 v1)**
- 입력은 **무료 일봉**(yfinance/Stooq 등)으로 받은 OHLCV DataFrame을 가정(엔진은 소스 무관, DataFrame만 소비).
- ⚠️ **생존편향 경고를 산출에 명기**: v1은 현재 유니버스+무료데이터라 생존편향 내장 → 결과는 **"낙관적 상한"**.
  **fail-fast 용도지 라이브 greenlight 아님**(라이브 전 생존편향 없는 벤더 재검증은 헌장 §3, 후속).

**④ LLM 격리**: 백테스트는 결정론적 알고리즘 엣지를 측정. 센티먼트/판단 provider는 주입형, 기본은
중립(통과) mock. LLM 실호출 금지(비결정성·미래참조).

**⑤ 워크포워드 / 인·아웃샘플 분리** — 헌장 §10 ②. train/test split 평가 함수 제공(커브피팅 킬러).

## 6. 산출 지표 (`BacktestResult` frozen dataclass) — 헌장 §6·§9·§10

- **`sharpe`(주 지표)**, `sortino`(보조), `max_drawdown`(분수), `total_return`, `cagr`, `profit_factor`, `expectancy`
- **SPY 벤치마크 대비**: 같은 기간 SPY 매수후보유의 Sharpe·CAGR·MDD와 **나란히 비교**(헌장 §9 — 승리=위험조정 우위).
- 거래 리스트(진입·청산·손익·사유), 레짐별 성과 분해.
- **이 `win_rate`·`win_loss_ratio`·`total_trades`가 `effective_kelly_fraction` 입력으로 직결**(켈리 연결).

## 7. 검증 테스트 (최소)

- 미래참조 차단(마지막 바 이후 변경→과거 불변), 결정론(2회 동일), 비용↑→수익·승률 단조 악화.
- 상승추세 합성데이터→양의 expectancy / 톱니(횡보)→저조. 레짐 D에서 신규진입 0.

- 0거래 안전(0분모 없이 win_rate=0 등). SPY 벤치마크 산출. 백테스트 결과→켈리 `[0,cap]` 산출.
