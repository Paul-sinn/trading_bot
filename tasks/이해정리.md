# 에이전트 이해 정리

알고리즘 3레이어 → LLM 판단 → 리스크 게이트 순서로 자동매매하는 봇.
서브에이전트 6개가 각자 주기로 도는 독립 루프이며, 모두 `AgentRegistry`에 등록되어
리스크 에이전트의 kill-switch로 일괄 정지될 수 있다.

> **워치리스트(watchlist)** = 봇이 감시할 종목 목록. 네가 "AAPL, TSLA, NVDA…" 처럼
> 관심 종목을 정해두면, 스캐너가 그 목록만 1분마다 돌며 매수 기회를 찾는다.
> (온 시장이 아니라 네가 지정한 종목들만 본다.)

## 서브에이전트 6개 (중요도순)

각 중요도%는 "봇의 핵심 목적 = 안전한 자동매매"에 얼마나 결정적인지의 비중.

| 에이전트 | 중요도 | 쉬운 설명 | 한 일 | LLM |
|---|---|---|---|---|
| **리스크** | **25%** | 안전벨트·비상정지 버튼. 사고 나면 다 멈춤 | 실시간 리스크 계산, 한도 초과 시 전 에이전트 강제 정지(kill-switch) | ❌ |
| **판단** | **25%** | 살까 말까 최종 결정하는 두뇌 | 후보를 종합 판단 → BUY/HOLD/SELL. 불확실하면 보수적 HOLD | ✅ |
| **실행** | **20%** | 실제로 주문 넣는 손. 진짜 돈이 움직임 | 리스크 게이트 통과 후 주문 실행, 체결 확인, 슬리피지 기록 | ❌ |
| **스캐너** | **15%** | 기회를 찾는 눈. 후보를 추려옴 | 워치리스트 1분 주기 스캔 → Layer1+Layer2 적용 → 후보 리스트업 | 간접(센티먼트) |
| **리포트** | **8%** | 성적표 작성. 거래엔 직접 영향 없음 | 일/주간 성과 집계(손익·승률·거래수) + AI 코멘트 → DB 저장 | ✅ |
| **알림** | **7%** | 알림 보내는 비서. 없어도 거래는 돎 | 체결·리스크·목표달성 이벤트 → 슬랙/SMS (채널 실패 격리) | ❌ |

**한 줄 비유:** 스캐너(눈)가 기회를 찾고 → 판단(두뇌)이 살지 정하고 → 실행(손)이 주문을 넣되,
→ 리스크(안전벨트)가 위험하면 다 멈추고 → 리포트(성적표)·알림(비서)이 결과를 남긴다.

## 매매 파이프라인 (한 종목이 매수까지)

```
워치리스트 종목
  │
  ▼ 스캐너 ── Layer1 시그널 (EMA/RSI/MACD 다수결)
  │        └ Layer2 필터 (거래량·ATR·뉴스센티먼트·VIX, 전부 AND)
  │           → 통과한 종목만 "후보"
  │
  ▼ 판단 ──── 뉴스+차트 종합 → BUY/HOLD/SELL
  │
  ▼ Layer3 포지션 사이징 (Kelly 변형, 하드캡)
  │
  ▼ 리스크 게이트 ── 통과해야 주문
  │
  ▼ 실행 → 체결 → 리포트 → 알림
```

## LLM 호출 지점 5곳

알고리즘은 순수 함수라 LLM을 직접 호출하지 않는다. LLM 호출은 모두 agents/backend에 있고,
provider 주입(`Mock*Provider` fallback + 실제 provider)으로 격리된다.

| # | 지점 | 위치 | 하는 일 |
|---|---|---|---|
| 1 | 센티먼트 | 스캐너 Layer2에 입력 | 뉴스/SNS → relevance/sentiment/market_impact/uncertainty 점수 |
| 2 | 판단 | `agents/decision.py` | 최종 BUY/HOLD/SELL 게이트 |
| 3 | 리포트 코멘트 | `agents/reporter.py` | 성과 통계 한줄평 |
| 4 | 시황 방향성 | `/direction` 백엔드 | 매일 9시 시황 요약 + 7일 방향성 |
| 5 | 목표 근거 | `backend/app/api/goal_plan.py` | 목표 역산 세팅의 "왜 이런가" 설명 |

## 안전 원칙 (LLM 관련)

- **LLM은 판단·설명·점수만 생성.** 리스크 한도·포지션 사이징 등 안전 수치는 알고리즘이 단일 진실이며
  LLM이 하드캡을 못 덮어쓴다 (ADR-003/005).
- **fail-closed:** 불확실하면 매매하지 않고 보수적 HOLD/스킵으로 떨어뜨린다.
- **센티먼트 LLM은 `algorithms/`에 두지 않는다.** agents/backend에서 호출해 점수만 필터에 인자로 넘긴다.
