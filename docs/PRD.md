# PRD: Custom Trading Bot

## 목표
알고리즘 시그널과 LLM(OpenAI) 최종 판단을 결합해, 개인 투자자의 Robinhood 계좌에서 리스크 한도 내 자동매매를 실행하고 실시간으로 모니터링하는 트레이딩 봇.

> LLM 제공자는 OpenAI다(ADR-007, 모델은 `OPENAI_MODEL` 환경변수, 기본 `gpt-4o`). 초기 설계는
> Claude 기준이었으나 운영 자격증명이 OpenAI 키라 교체했다. LLM은 설명·판단만 하고 리스크 수치는
> 알고리즘이 단일 진실이다(ADR-003/005). 키 부재 시 Mock으로 안전 fallback.

## 사용자
- Robinhood 계좌를 보유한 개인 트레이더 (1인 운영, 본인 계좌).
- 매일 차트를 직접 보지 않아도 규칙 기반 + AI 검토로 매매를 위임하고 싶은 사용자.
- 손실 한도와 투자성향을 직접 설정하고, 봇의 판단 근거를 확인하려는 사용자.

## 핵심 기능
1. **실시간 포트폴리오 동기화** — Robinhood MCP → WebSocket → UI 1초 갱신 (총자산, 손익, 포지션).
2. **자동 주문 실행** — 알고리즘 3레이어 통과 → LLM(OpenAI) 최종 검토 → MCP 주문 실행.
3. **리스크 자동 차단 (kill-switch)** — 실시간 포트폴리오 손실 한도 초과 시 모든 에이전트 강제 정지 + 알림 발송.
4. **AI 시황 분석** — 매일 9시 LLM(OpenAI)이 시황 요약 및 다음 7일 방향성 라벨(강세/중립/약세) + 근거 생성.
5. **수동 직접거래** — UI에서 티커 검색 → 수량 입력 → 1클릭 매수/매도.
6. **투자성향 반영** — 공격적↔보수적 슬라이더 변경 시 포지션 크기 & 스탑로스 자동 재계산.

## 알고리즘 (3레이어)
- **Layer 1 시그널**: EMA 크로스(9/21), RSI(14) 70/30, MACD 히스토그램 방향. 지표는 TA-lib가 아니라 pandas/numpy로 직접 계산(환경 의존성 제거).
- **Layer 2 필터**: 거래량 급등, ATR 변동성 필터, 뉴스 센티먼트(LLM/OpenAI), VIX 체크.
- **Layer 3 사이징**: fractional Kelly(비례 축소) + 하드캡 상한, 스탑로스 = 진입가 −(ATR×배수), 투자성향 가중치, 매매당 최대 리스크% 한도.
  - Kelly 입력(승률·손익비)은 백테스트/거래기록에서 추정한다. 거래기록이 없는 콜드스타트에는 Kelly를 신뢰하지 않고 보수적 고정 비율로 시작해, 표본이 쌓이면 점진 전환한다(shrinkage).
- 3레이어 모두 통과한 종목만 LLM(OpenAI) 판단 에이전트의 최종 승인을 받는다.

## 서브에이전트 (6개)
1. **스캐너** — 워치리스트 순환 스캔, 시그널 후보 리스트업 (1분 주기).
2. **판단** — LLM(OpenAI API) 호출, 뉴스+차트 종합, 매수/홀드/매도 결정.
3. **실행** — MCP 주문 실행, 체결 확인, 슬리피지 기록.
4. **리스크** — 실시간 리스크% 계산, 한도 초과 시 전 에이전트 kill-switch.
5. **리포트** — 일간/주간 성과 집계, AI 코멘트, DB 저장.
6. **알림** — 체결·리스크·목표달성 이벤트 → 슬랙/SMS.

## MVP 제외 사항
- 멀티 계좌 / 멀티 유저 (1인 본인 계좌 전용).
- 옵션·선물·암호화폐 (주식 현물만).
- 모바일 네이티브 앱 (반응형 웹만).
- PostgreSQL/Redis 프로덕션 인프라는 후순위 (개발은 SQLite, 캐시는 후속 단계).

## 계획됨 (MVP 직후 우선 작업)
- **백테스트 엔진** (`algorithms/backtest.py`): 3레이어 전략을 과거 데이터에 재생해 승률·손익비·기대값·MDD를 산출. Kelly 사이징의 입력(콜드스타트 prior)과 엣지 검증의 근거가 되므로 사실상 1순위. (당초 "제외"였으나 Kelly가 추측이 되지 않도록 범위에 포함.)

## 디자인
- 다크모드 고정. "매일 쓰는 트레이딩 도구"로서의 대시보드 (마케팅 페이지 아님).
- 무채색 베이스 + 손익 시맨틱 색상(상승 녹색 / 하락 적색)만 포인트.
- 데이터 밀도 우선, 장식 최소화. 차트는 Recharts.
